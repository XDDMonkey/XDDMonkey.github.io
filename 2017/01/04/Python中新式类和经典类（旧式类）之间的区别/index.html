Python中新式类和经典类（旧式类）之间的区别

发表于 2017-01-04 |

　　其实因为现在Python中使用的都是新式类了，讨论新式类和经典类的意义也不是很大了，但是按捺不住自己的好奇心，还是小小总结一下，思来想去，还是有很多不完备的地方，按照《Python学习手册》上的内容来细分了一下，重点分析了其中的一些点，暂有如下内容，随时更新：

新式类和经典类的背景

新式类的提出时间

　　新式类是在Python2.2开始提出和引入的概念（因此在下文中，如果有提到Python2中的新式类，自然是指Python2.2之后），在Python2.2之前type（类型）和class（类）是两个不同的概念（为什么要有类型，自己大胆猜测是不是在Python2.2以前，内置类型可能不是类，而仅仅是一种类型？），因此在经典类中，N().__class__和type(N())返回的结果是不一样的（至于为什么之前要设置成不一样的，自己还没有找到有力的资料）。但是在提出了新式类之后，对于任何的新式类N，N().__class__和type(N())返回的结果通常将是一样的（当然我们无法保证这一点，因为新式类的实例的__class__方法是允许被用户覆盖的），与此同时，旧式类在python3中被移除，只留下了新式类。
　　截止到python2.1，只存在旧式类，在旧式类中，类名和type是无关的，如果x是一个旧式类，那么x.__class__定义了x的类名，但是type(x)总是返回。这反映了所有的就是类的实例是通过一个单一的叫做instance的内建类型来实现的，这是它和类不同的地方。

Python2和Python3的新式类

　　Python2中用户创建的类默认都是经典类，只有显式继承了object的才是新式类，更加准确的情况是由任意内置类型派生出来的类（只要有一个内置类型位于类树的某个位置上），就属于新式类，当然内置类型本身的类也是属于新式类了。
　　在Python2中，如果我们想要定义一个新式类，我们可以让它的类树上有内置类型派生出来的类就可以了，但是如果我们不想让他继承任何一个类的时候，我们就可以让它继承自object。
　　Python3中默认都是新式类，不必显式继承，默认都是新式类。

主要区别

类和类型的合并

　　类和类型的合并就带来了__class__和type()的合并。
　　新式类可以直接通过__class__属性获取自身type

《Python学习手册》P1054：Python3.0中的类（以及Python2中2.6及其以后的新式类）是type类的实例，并且实例对象是他们的类的实例。实际上，类现在连接到了type的一个__class__，就像是一个实例有了连接到创建它的类的__class__。

　　关于传统类，书上也有叙述：

《Python学习手册》P1055：然而，Python2.6中的传统类略有不同——因为它们反映了老Python版本中的类模型，它们没有一个__class__链接，并且像Python2.6中的内置类型一样，它们是type的实例，而不是一个类型实例。

　　关于这里的type的实例，而不是类型的实例，自己没有怎么看明白，还存在一定疑问。
　　上述书中的叙述，自己可以总结一下：
　　1. 新式类都是继承object的，经典类中没有这个特性。
　　2.对于经典类，a.__class__和type(a)都可以查看对象模型，但是显示不一样的结果，这一点在新式类中进行了修正。

继承搜索顺序

《Python学习手册》P780：多继承的钻石模式有一种略微不同的搜索顺序，总体而言，它们可能先横向搜索再纵向搜索，并且先宽度优先搜索，再深度优先搜索。

　　实际上翻译过来的规则就是：
　　1.如果第一个序列的第一个元素，是后续某个序列中的第一个元素，或者在后续的序列中不再出现，则这个元素合并到最终的方法解析顺序序列中，并从当前操作的全部序列中删除。
　　2.如果不符合（即如果在后续序列中出现了，但是并不是第一个），则跳过此元素，查找下一个列表的第一个元素，重复1的判断规则。
那么对于D和E，则有：
　　mro(D) = [D] + merge(mro(A), mro(B), [A, B])
　　= [D] + merge([A, O], [B, O], [A, B])
　　= [D, A] + merge([O], [B, O], [B])
　　= [D, A, B] + merge([O], [O])
　　= [D, A, B, O]
　　mro(E) = [E] + merge(mro(C), mro(D), [C, D])
　　= [E] + merge([C, O], [D, A, B, O], [C, D])
　　= [E, C] + merge([O], [D, A, B, O], [D])
　　= [E, C, D] + merge([O], [A, B, O])
　　= [E, C, D, A, B] + merge([O], [O])
　　= [E, C, D, A, B, O]

针对内置函数的属性获取

《Python学习手册》P780：__getattr__和__getattribure__方法不再针对内置运算的隐式属性获取而运行。这意味着，它们不再针对__X__运算符重载方法名而调用，这样的名称搜索从类开始，而不是从实例开始。

新的高级工具

《Python学习手册》P780：新式类有一组新的类工具，包括slot、特性、描述符和__getattribute__方法。这写工具中的大多数都有非常特定的工具构建目的。
